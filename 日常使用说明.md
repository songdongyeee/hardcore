# 双引擎系统使用说明

## 🖥️ 终端窗口在哪？

**在 VS Code 中**：
- 底部的 "终端" (Terminal) 面板
- 应该有一个标签正在运行 `python3 scripts/mac_whisper_worker_v2.py`
- 你会看到实时的日志输出

**如果看不到终端**：
- 按快捷键：`` Ctrl + ` `` (反引号)
- 或者：菜单 → 查看 → 终端

---

## 📤 以后上传材料怎么办？

**不需要告诉我！** Worker 会自动处理，完全自动化。

### 日常使用流程

#### 方式 1: 你上传材料（用 Whisper）

1. **启动 Worker**（第一次）
   ```bash
   cd /Users/geo/Downloads/hardcoreEnglish
   python3 scripts/mac_whisper_worker_v2.py
   ```
   保持这个终端窗口开着

2. **登录 PB 后台**
   https://zjcnex.top/_/

3. **上传材料**
   - Collections → transcripts → New record
   - 上传音频
   - **设置 `asr_engine = whisper`** ⭐
   - 保存

4. **Worker 自动处理**
   - 自动检测（每 10 秒一次）
   - 自动下载音频
   - 自动 Whisper 转写
   - 自动翻译
   - 自动上传结果
   - **你什么都不用做！**

#### 方式 2: 用户 App 上传（用阿里云）

用户在 App 上传，自动使用阿里云，不经过 Worker。

## 💡 日常使用技巧

### Worker 何时需要运行？

**只有在你要上传材料时才需要开着 Worker**

- 上传前启动 Worker
- 上传后材料会自动处理
- 处理完成后可以关闭 Worker（Ctrl+C）
- 用户上传时不需要 Worker

## ❓ 常见问题

**Q: Worker 必须一直开着吗？**
A: 不需要。只有你要上传材料时才开。用户 App 上传不需要 Worker。

**Q: Worker 关闭了，之前的任务会丢失吗？**
A: 不会。重新启动 Worker，未完成的任务会继续处理。

**Q: 怎么知道转写完成了？**
A: 看 Worker 日志，或者刷新 PB 后台，`status` 变成 `done`。

**Q: 可以同时转写多个吗？**
A: 当前是一个一个处理。可以上传多个，Worker 会自动排队处理。

**Q: 转写很慢怎么办？**
A: 这是正常的。10 分钟音频可能需要 5-8 分钟转写。你可以上传后去做其他事，Worker 会自动完成。
