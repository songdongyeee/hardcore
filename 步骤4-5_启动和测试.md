# 步骤 4-5: 启动 Worker 并测试

## 步骤 4: 启动 Mac Worker

### 启动命令

在终端运行：
```bash
cd /Users/geo/Downloads/hardcoreEnglish
python3 scripts/mac_whisper_worker.py
```

```

[20:07:00] 📍 PocketBase: https://zjcnex.top
[20:07:00] ⏰ 检查间隔: 10秒

[20:07:01] 👀 开始监听转写任务...
```

## 步骤 5: 测试转写

### 测试 Whisper 模式（你的后台上传）

1. **打开新终端/浏览器标签**（不要关闭 Worker）

2. **登录 PB 后台**
   - https://zjcnex.top/_/

3. **创建测试记录**
   - Collections → transcripts → New record
   
4. **填写字段**：
   - `audio`: 上传一个**短音频**（1分钟以内，测试用）
   - `language`: `en`
   - **`asr_engine`**: `whisper` ⭐⭐⭐ 重要！
   - `status`: 留空（会自动设置）

5. **保存**

6. **观察 Worker 终端**
   你会看到：
   ```
   [20:08:15] 📬 发现 1 个待处理任务
   [20:08:15] ==================================================
   [20:08:15] 📋 处理任务: xyz123abc
   [20:08:16] 📥 音频已下载: test.mp3
   [20:08:16] 🎙️ 开始 Whisper 转写...
   [20:08:35] ✅ 转写完成！生成 5 个句子
   [20:08:35] 🔄 调用阿里云翻译...
   [20:08:40] ✅ 翻译完成
   [20:08:41] ✅ 结果已上传到服务器
   [20:08:41] 🎉 任务完成: xyz123abc
   ```

7. **检查结果**
   - 刷新 PB 后台的 transcripts 列表
   - 找到刚才的记录
   - `status` 应该变成 `done`
   - 点击查看 `text` 字段，应该有完整的 JSON

---

### 测试阿里云模式（用户 App 上传）

为了验证用户上传不受影响：

1. **创建另一个测试记录**
2. 上传音频
3. **`asr_engine`**: 留空或选 `aliyun`
4. 保存

应该会立即被阿里云处理（不经过 Worker）
